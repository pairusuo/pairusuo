# 做人

## 人生态度

- 要么做生产者，要么做消费者，生产者对应赚钱，消费者对应花钱。
- 人一生几十年，会遇到各种大灾难，大机会；经济也有上行期下行期，行业还有康波周期。所以不存在悲观乐观，只存在身份的切换，能做什么就做什么。
- 在大趋势面前个人无能为力的时候（比如 AGI 浪潮），停下来好好享受生活也不是不行，不是非要出人头地。
- 不要和消耗你的人玩，要和滋养你的人玩。

# 挣钱

## 上班

- 上班只能赚钱，不能挣钱。在公司，员工用时间换取金钱，这是等价交换。而挣钱（创业）需要个人冒着极大的风险和投入大量的时间精力，才有可能获得财富。所以上班挣不到钱。
- 我们不能长期单打独斗，要通过和别人合作来挣钱。

## 炒股

- 每次股票交易时都想一下，自己是在赚什么钱。如果是赚波动的钱，那涨了就可以卖，不要没卖然后被套，又安慰自己说是价值投资，长期持有。
- 期权，以小博大的机会，可以了解一下。

## 净资产

- 消费金额不足净资产万分之一的话，不值得花时间比较价格。比如我净资产 500 万，500 以下的东西根本没必要去比价，买。
- 能让净资产增加百分之一的事情，都值得花时间去研究。

## 做网站

- 逆向是个好思路，你想要什么效果，就截图给 AI，让 AI 总结，然后让 AI 出一份提示词，再根据提示词生成效果并让 AI 继续优化。

### API 网关

- 有多个网站的话，分散在各处的 API Key 如何管理？当前是用本地密码工具记录，但我感觉终极方案是 API 网关，在服务一端进行统一管理，这样既安全又方便。

### 域名关键词

- 做品牌站，域名关键词要短，甚至可以造新词，比如我的 AI Omnify，其实可以注册域名 omniai.com 这种，而我没有造新词，而是使用了 Omnify 这个已有的单词，但是用户搜这个单词的搜索意图和我的网站定位不一致。不过现实就是 omniai 这个关键词的域名也早就被抢完了，但这不是重点，重点是做品牌站就要敢于造新词。

### 找词

- 找不到词的时候可以多看看别人的网站，在可灵这类平台上有很多素人做的轻应用，有些想法还不错，这些轻应用也能帮忙验证市场。

### 外链

- 子域名权重为零，没必要在使用子域名的网站里做外链。

### 他山之石

- 域名拼错了不要慌，自己会犯错，用户也会犯错，错误的域名也能做 SEO（FAQ 和内页可以承接错别字带来的流量），再注册一个正确的域名，做个跳转。
- 做的网站如果有用户交互的话，注意 UGC 内容不要影响了主关键词密度。
- 多语言不能机器翻译随便糊弄（那怎么弄呢？），hreflang 和 sitemap 要正确配置。
- TDH 第一版就要确定好，后续不要轻易修改，改坏的概率更大。icon 要在第一版就确定下来，后续修改的话，Google 更新会很慢。
- 要准确理解用户搜索意图，观察用户时区，有针对性地做运营。
- 移动端样式很重要，它是 Google 判断移动友好性的因素之一，即使移动端用户不多。
- 不要只盯着 Google，Yandex 和 Bing 也需要提交。

## 作图

- 柱状图能直观地看出占比，但是气泡图才能直观地看出规模，柱状图想要看出规模的话，还要结合纵轴。

## 做视频

- 使用 AI 做视频时，其实是需要先生成多个分镜图，然后根据分镜图生成视频，效果应该比直接文生视频要好。

## 做产品

- AI 时代，SaaS 还是有需求，不过 SaaS 是给人用的，还是传统的 UI 交互；Agent 也是给人用的，只是没有几乎 UI，因为不需要。两个形态的产品都要做。
- 工具类的网站，最后可能都会被大模型公司出的 app 取代，毕竟你在这些 app 里的一句 prompt 就是一个工具，就能实现你的需求，可能不是实现得最好的那个，但是够用，且大模型会让其越来越好，最重要的是更易用了。比如文生图，以前还要 prompt 技巧，现在直接自然语言就能生成不错的图片。所以做工具也要趁早，要不就得提供工具之外的价值，比如情绪价值，所以做电子宠物反而比做工具网站更有前途，除非你的工具壁垒很高。

- 是否有必要做一个国内的 VPN，按城市区分节点？比如人在武汉，但是我想让抖音认为我在深圳，从而推流深圳的内容给我。
