## 1. 段落与换行

- 空一行开始新段落。
- 行内强制换行：行尾两个空格，或显式 `<br />`。

```md
第一行  ←← 行尾两个空格  
第二行

这是新段落
```

渲染：
第一行  
第二行

这是新段落

## 2. 标题（Headings）

使用 `#` 到 `######` 创建 1–6 级标题，`#` 后建议加空格。

```md
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

- 本站开启了自动目录锚点，可点击标题左侧/右侧的链接跳转。

## 3. 强调（Emphasis）

- 斜体：`*text*` 或 `_text_` → *text*
- 粗体：`**text**` 或 `__text__` → **text**
- 斜粗体：`***text***` → ***text***
- 删除线（GFM）：`~~text~~` → ~~text~~
- 行内代码：`` `code` `` → `code`

```md
这是 *italic*、**bold**、***bold italic***、~~strike~~，以及 `inline code`。
```

## 4. 引用（Blockquote）

以 `>` 开头，可多层嵌套。

```md
> 这是引用段落。
> 
> - 也可以包含列表
> - 或其他 Markdown 元素
>> 二级引用
```

## 5. 列表（Lists）

- 无序列表：`-` / `*` / `+`
- 有序列表：`1.`、`2.`（数字会自动递增）
- 嵌套：缩进两个或四个空格

```md
- 项目 A
- 项目 B
  - 子项 B1
  - 子项 B2

1. 步骤一
2. 步骤二
   1. 子步骤 2.1
   2. 子步骤 2.2
```

提示：列表项之间保持空行可读性更佳；如遇段落换行，子段落需继续缩进与父项对齐。

## 6. 代码（Code）

- 行内代码：使用单个反引号 `` ` ``
- 代码块：使用三个反引号，建议标注语言实现高亮

```ts
export function greet(name: string) {
  return `Hello, ${name}!`;
}
```

```bash
# 命令行示例
npm run dev
```

> 注意：若代码中本身包含三个反引号，可用更长的围栏如 ```` ```` 包裹。

## 8. 链接与图片（Links & Images）

- 行内链接：`[文本](URL)`
- 行内图片：`![替代文本](URL "可选标题")`
- 自动链接（GFM）：裸露的 `https://example.com` 会自动转为链接

```md
[Next.js 官网](https://nextjs.org/)

![示例图片](/images/example.png "示例标题")

访问 https://markdown.com.cn 获取更多内容
```

> 图片路径推荐放在 `public/` 目录，例如 `/images/...`。

### 8.1 参考式链接（可读性更好）

```md
这是一个[参考式链接][next]，也可以简写为[Google][]。

[next]: https://nextjs.org/
[Google]: https://www.google.com
```

## 9. 转义（Escapes）

要显示 Markdown 字面量，用反斜杠 `\` 转义：

```md
\*不是强调\*，而是字面星号。
```

渲染：\*不是强调\*。

## 10. 表格（Tables，GFM）

在 `remark-gfm` 下支持表格与对齐：

```md
| 字段 | 类型 | 说明 |
| :--- | :--: | ---: |
| name | str  | 右对齐 |
| age  | int  | 右对齐 |
```

| 字段 | 类型 | 说明 |
| :--- | :--: | ---: |
| name | str  | 右对齐 |
| age  | int  | 右对齐 |

## 11. 任务列表（Task List，GFM）


## 12. 脚注（Footnotes，GFM 扩展）

```md
支持脚注示例[^1]，可在文末定义。

[^1]: 这是脚注内容，支持多段落与 Markdown。
```

支持脚注示例[^1]。

[^1]: 这是脚注内容，支持多段落与 Markdown。

## 13. HTML 与 MDX

- 标准 Markdown 可内嵌少量 HTML 标签：`<br />`、`<sub>`、`<sup>` 等。
- 本站支持 MDX：如需交互组件，请使用客户端组件并在文件顶部或组件内添加 `"use client"`。

> 建议文章以 Markdown 为主，MDX 组件谨慎使用，以保证可读性与可维护性。

## 14. 常见坑与最佳实践

- 标题 `#` 后加空格；中文标点前后保持一致空格习惯。
- 列表嵌套时，子元素缩进与父项对齐；列表项内新段落需继续缩进。
- 图片加 `alt` 文本以提升无障碍与 SEO。
- 大段代码务必使用代码块并指明语言，便于高亮与复制。
- 链接尽量使用参考式，方便统一管理。
