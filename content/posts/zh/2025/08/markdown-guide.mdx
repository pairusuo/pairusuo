> 本文演示如何用 Markdown/MDX 在本博客写作。保存或发布后，系统会自动按“年/月/slug”生成目录，并使用东八区完整时间戳。

## 1. 标题（Headings）

使用 `#` 到 `######` 创建 1~6 级标题：

```md
# 一级标题
## 二级标题
### 三级标题
```

## 2. 强调（Emphasis）

- 斜体：`*text*` 或 `_text_` → *text*
- 粗体：`**text**` 或 `__text__` → **text**
- 删除线：`~~text~~` → ~~text~~
- 行内代码：`` `code` `` → `code`

## 3. 段落与换行

空一行开始新段落。行尾加两个空格或显式 `<br />` 可强制换行。

## 4. 列表（Lists）

- 无序列表：使用 `-` / `*` / `+`
- 有序列表：使用 `1.`、`2.` …（自动递增）

```md
- 项目 A
- 项目 B
  - 子项 B1

1. 步骤一
2. 步骤二
```

## 5. 引用（Blockquote）

使用 `>`：

```md
> 这是引用段落。
>> 支持嵌套引用。
```

## 6. 链接与图片（Links & Images）

- 链接：`[文本](https://example.com)`
- 图片：`![替代文本](/images/example.png)`（推荐放到 `public/` 下，如 `public/images/...`）

```md
[Next.js 官网](https://nextjs.org/)

![示例图片](/images/example.png)
```

## 7. 代码（Code）

- 行内代码：使用反引号 `` `code` ``
- 代码块：使用三反引号并指定语言高亮（已启用 Shiki）

```ts
export function hello(name: string) {
  return `Hello, ${name}!`;
}
```

```bash
# 命令行示例
npm run dev
```

> **提示**：发布/保存草稿时，slug 无需包含 `/`，系统会自动加 `YYYY/MM/` 前缀。

## 12. MDX 注意事项

- 本站支持 MDX，可在文中嵌入 React 组件（如需）。
- 组件需确保是客户端组件（如带交互）时添加 `"use client"`，并注意在 MDX 中导入/使用方式。
- 一般文章只用 Markdown 即可，避免过度复杂的组件以提升可读性与可维护性。

## 13. Frontmatter 规范

文章顶部使用 YAML 作为元信息：

```md
---
title: 文章标题
summary: 文章摘要
tags: [tag1, tag2]
publishedAt: 2025-08-10 21:14:36
updatedAt: 2025-08-10 21:14:36
draft: false
---
```

- `publishedAt`/`updatedAt`：统一使用上海时区完整时间 `YYYY-MM-DD HH:mm:ss`
- `draft`：草稿为 `true`；发布后写入 `false`
- slug：在 Admin 中只填短 slug（小写字母/数字/中划线），系统会自动加 `YYYY/MM/` 目录。

## 14. 预览与发布

- 在 Admin 页面可先保存为草稿，草稿列表支持刷新与一键发布。
- 发布成功后将跳转到对应文章页，路径：`/blog/YYYY/MM/slug`（英文为 `/en/blog/YYYY/MM/slug`）。
